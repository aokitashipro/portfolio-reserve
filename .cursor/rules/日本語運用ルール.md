# 日本語運用ルール - Cursor AI

## 🌐 言語設定（最重要）

### ✅ すべて日本語で書くもの

1. **コミットメッセージ**: 必ず日本語
2. **PR（Pull Request）タイトル**: 必ず日本語
3. **PR説明文**: 必ず日本語
4. **GitHub Issue**: 必ず日本語
5. **コードコメント**: 日本語推奨

### ✅ 英語で書くもの

1. **ブランチ名**: 英語のみ（例: `feature/user-authentication`）
2. **変数名・関数名**: 英語のみ（例: `getUserById`）
3. **ファイル名**: 英語のみ（例: `BookingForm.tsx`）

---

## 📝 コミットメッセージ（日本語）

### フォーマット

```
<プレフィックス>: <変更内容>

<詳細説明>

<関連Issue>
```

### 例

```bash
feat: ユーザー登録機能を実装

- Supabase Auth統合
- Zodバリデーション追加
- E2Eテスト作成

Closes #5
```

```bash
fix: 予約カレンダーの日付選択バグを修正

過去の日付がクリックできてしまう問題を修正。
disabledプロパティを追加。

Closes #12
```

```bash
test: 管理者ダッシュボードのE2Eテストを追加
```

```bash
docs: READMEにセットアップ手順を追記
```

---

## 🔀 PR（Pull Request）タイトル（日本語）

### フォーマット

```
[プレフィックス] 変更内容の要約
```

### 例

```
[FEATURE] ユーザー登録機能を実装
[FIX] 予約カレンダーのバグを修正
[TEST] E2Eテストを追加
[DOCS] API仕様書を作成
```

---

## 📋 PR説明文（日本語）

### テンプレート

```markdown
## 📝 概要
ユーザー登録機能を実装しました。

## 🎯 関連Issue
Closes #5

## ✅ 実装内容
- [x] 登録フォームコンポーネント作成
- [x] Supabase Auth統合
- [x] Zodバリデーション実装
- [x] E2Eテスト作成

## 🧪 テスト方法
1. `npm run dev` で開発サーバー起動
2. http://localhost:3000/register にアクセス
3. フォームに入力して送信

## 📊 品質チェック
- [x] ESLintエラー0件
- [x] TypeScriptエラー0件
- [x] テスト通過
- [x] E2E通過
```

## 🎯 プレフィックス一覧

| プレフィックス | コミット | PR |
|------------|---------|-----|

| バグ修正 | `fix:` | `[FIX]` |
| テスト | `test:` | `[TEST]` |
| リファクタリング | `refactor:` | `[REFACTOR]` |
